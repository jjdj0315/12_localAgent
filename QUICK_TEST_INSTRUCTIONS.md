### 현재 상태
- ✅ 백엔드: http://localhost:8000 (정상)

## 📝 Step 1: 브라우저에서 접속

### 1-1. 브라우저 열기
```
http://localhost:3000
```

### 1-2. 로그인
- **사용자명**: `admin`
- **비밀번호**: (초기 설정 시 입력한 비밀번호)

만약 로그인이 안 된다면:
- 백엔드 로그 확인: `docker-compose logs backend | tail -50`
- 사용자 확인: 이미 admin 계정이 생성되어 있습니다

## 📊 Step 2: 관리자 대시보드 접속

로그인 후 자동으로 `/admin` 페이지로 이동하거나, 직접 접속:
```
http://localhost:3000/admin
```

**예상되는 화면**:
- 상단: 관리자 대시보드 제목
- 섹션들: 사용자 관리, 시스템 정보, **시스템 메트릭 히스토리** 등

## 🎯 Step 3: 핵심 기능 빠른 확인 (5분)

### ✅ 1. 메트릭 그래프 확인 (AS-1.1)
- [ ] **6개 그래프**가 표시되는지 확인
  - 활성 사용자 수
  - 저장소 용량 (바이트)
  - 활성 세션 수
  - 대화 개수
  - 문서 개수
  - 태그 개수

### ✅ 2. 그래프 인터랙션 (AS-1.2)
- [ ] 그래프 위에 **마우스 호버** → 툴팁이 한글로 표시되는지 확인
- [ ] 시간과 값이 표시되는지 확인

### ✅ 3. 시간 범위 선택 (AS-1.3)
- [ ] **7일 / 30일 / 90일** 버튼 클릭 → 그래프 변경 확인
- [ ] **시간별 / 일별** 토글 → 데이터 포인트 변경 확인

### ✅ 4. 비교 모드 (AS-2.1)
- [ ] **"비교"** 버튼 클릭
- [ ] **"이번 주" / "지난 주"** 선택
- [ ] 두 기간이 **다른 색상**으로 오버레이되는지 확인

### ✅ 5. CSV 내보내기 (AS-3.1) ⭐
- [ ] **"CSV 내보내기"** 버튼 클릭
- [ ] 파일 다운로드 확인
- [ ] Excel에서 열어서 **한글 헤더** 확인

### ✅ 6. PDF 내보내기 (AS-3.3) ⭐⭐⭐ **가장 중요!**
- [ ] **"PDF 내보내기"** 버튼 클릭
- [ ] PDF 파일 다운로드 확인
- [ ] PDF 파일 열기
- [ ] **한글 폰트가 정상적으로 렌더링**되는지 확인
  - ✅ 정상: "활성 사용자 수", "저장소 용량" 등이 제대로 보임
  - ❌ 실패: 네모 박스(□□□) 또는 깨진 글자

### ✅ 7. 수집 상태 표시기 (FR-018)
- [ ] **"데이터 수집 상태"** 위젯 확인
- [ ] 색상 확인:
  - 녹색 (정상) / 노란색 (주의) / 빨간색 (오류)
- [ ] 마지막 수집 시간 표시 확인
- [ ] 실패 카운트 표시 확인

## 🐛 문제 발생 시

### 로그인이 안 됨
```bash
# 관리자 계정 확인
docker exec llm-webapp-postgres psql -U llm_app -d llm_webapp -c "SELECT username, is_admin FROM users WHERE is_admin = true;"
```

### 그래프가 안 보임
```bash
# 프론트엔드 로그 확인
docker-compose logs frontend --tail 50

# 백엔드 로그 확인
docker-compose logs backend --tail 50
```

### 메트릭 데이터 없음
```bash
# 메트릭 데이터 확인
docker exec llm-webapp-postgres psql -U llm_app -d llm_webapp -c "SELECT metric_type, COUNT(*) FROM metric_snapshots GROUP BY metric_type;"
```
